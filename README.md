## Overview 

This repository contains all code, style definitions and cached numerical
outputs needed to reproduce every figure and numerical result in the article. It
is the reproducibility archive cited via its Zenodo DOI in the paper's reference
list. The work reviews relativistic statistical distributions — from the
Maxwell–Jüttner distribution through the quantum (Fermi–Dirac / Bose–Einstein)
statistics and their non-extensive Tsallis-*q* and Kaniadakis-*κ* generalizations,
the thermodynamic (Fisher–Rao / Ruppeiner) geometry of these ensembles, and a
Bayesian blast-wave inference pipeline (emcee MCMC with a Gaussian-process
model-discrepancy term) applied to relativistic heavy-ion spectra. Each figure is
generated by a single, self-contained script that imports one shared style sheet
and one shared physics module, so the entire figure set regenerates with a single
command.

The code reflects the **corrected results** of the final manuscript:

* the **master occupation** in its corrected form
  `n = 1/([exp_σ(−β(ε−μ))]⁻¹ + a)` (the reciprocal of the deformed exponential),
  so the Tsallis classical branch carries the power-law tail `ε^{−1/(q−1)}`
  (`_common.exp_q`, `figures/fig13_family_tree.py`, `figures/fig06_ricci.py`);
* the **relativistic BEC critical temperature** is *monotonic*: `T_c(m)` diverges
  as `m → 0` like `m^{−1/2}` (there is **no** mass-independent "398 MeV plateau")
  and lies below the non-relativistic extrapolation
  (`calculations/bec_critical_temperature.py`, `figures/fig11_bec_tc.py`);
* the **thermal conductivity** is the corrected positive **Schur complement**
  `κ_th ∝ K⁽⁴¹⁾ − (K⁽³¹⁾)²/K⁽²¹⁾ = Var_w(E) ≥ 0`, replacing the identically-zero
  bracket of an earlier draft (`calculations/transport_coefficients.py`).

## Directory layout

```
code/
├── README.md              # this file
├── LICENSE                # MIT
├── CITATION.cff           # machine-readable citation metadata (+ ORCIDs, DOI)
├── requirements.txt       # pinned, tested dependency versions (only what is imported)
├── .gitignore             # figures/*.pdf|png are deliverables (NOT ignored)
├── style.mplstyle         # canonical Matplotlib style (serif, CB-safe, 300 dpi)
├── _common.py             # shared constants + physics helpers + savefig_dual()
├── make_all.py            # runner: discovers & times every figure/calc script
├── figures/               # figNN_*.py  ->  <name>.pdf (vector) + <name>.png
├── calculations/          # analytic / numeric verification scripts (Block A/B/E)
├── bayesian/              # BGBW forward model + emcee driver (Block C)
├── pinn/                  # pinn_tsallis_boltzmann.py PINN prototype (Block D)
└── data/                  # cached numerical outputs (.npz): MCMC chain, BEC Tc,
                           #   PINN history, transport table (regenerable)
```

`data/` holds **cached, regenerable** numerical products (`.npz`), not
redistributed experimental tables: the real ALICE spectra are fetched on demand
from HEPData by the scripts that need them (see provenance below).

## Installation

Requires **Python 3.13** (developed/tested on 3.13.2). Install the pinned, tested
dependencies into a fresh virtual environment:

```bash
python -m venv .venv
# Windows:  .venv\Scripts\activate     Linux/macOS:  source .venv/bin/activate
pip install -r requirements.txt
```

The versions in `requirements.txt` are exactly those the results were produced
with: numpy 2.4.4, scipy 1.17.1, matplotlib 3.10.9, emcee 3.1.6, corner 2.3.0,
torch 2.12.0 (CPU wheel `2.12.0+cpu`). Only packages actually imported by the
scripts are listed. The Gaussian-process model-discrepancy term of the BGBW
likelihood is implemented in pure numpy/scipy (no scikit-learn); HEPData tables
are fetched with the standard library (`urllib` + `json`, no h5py).

## Reproducing all figures

From the `code/` directory:

```bash
python make_all.py            # discover, run and time every figNN_*.py + calc
python make_all.py --list     # dry run: list what would be executed
python make_all.py figures    # restrict to the figures/ group only
```

`make_all.py` runs each script in an isolated subprocess, prints a status/timing
summary table, and exits non-zero if any script fails (suitable for CI). It
discovers `figures/fig*.py` and `calculations/*.py`. Every figure script also
runs standalone, e.g. `python figures/fig01_maxwell_juttner.py`, and writes
**both** a vector `figures/<name>.pdf` (for Elsevier) and a raster
`figures/<name>.png` (preview) at 300 dpi via `_common.savefig_dual`.

**Cached heavy run.** The genuine ~14-minute emcee MCMC fit,
`bayesian/run_emcee_bgbw.py`, is intentionally **not** in the `fig*.py` glob and
is **not** re-run by `make_all.py`. It produces the cached chain
`data/bgbw_chain.npz`, which the figures (`fig07`, `fig07b`) read back. To
regenerate the chain from scratch (real ALICE data → labelled synthetic
closure-test fallback if HEPData is unreachable):

```bash
python bayesian/run_emcee_bgbw.py     # ~14 min, writes data/bgbw_chain.npz
```

The cached chain shipped here was produced from **real ALICE data**
(`SYNTHETIC_CLOSURE_TEST = False`). The `calculations/*.py` scripts write their
own caches (`bec_tc.npz`, `transport_coefficients.npz`); `fig11`/`fig12` will run
their upstream calculation automatically if the cache is missing.

## Heavy runs

The light figures (0–6, 11, 13, 14) compute locally in seconds. The heavier
pipelines are:

* the **emcee** MCMC blast-wave fit (`bayesian/run_emcee_bgbw.py`, ~14 min on one
  core) feeding Figs. 7 and 7b — cached in `data/bgbw_chain.npz`;
* the **PINN** training (`pinn/pinn_tsallis_boltzmann.py`, a couple of minutes on
  CPU) feeding Fig. 12 — cached in `data/pinn_history.npz`.

Both can be run on a free-GPU **Kaggle Notebook**; on Kaggle `torch` may resolve
to a CUDA build rather than the CPU wheel pinned in `requirements.txt`. Any
Kaggle/HEPData credentials must be supplied out-of-band and **never** committed or
printed — they are excluded in `.gitignore`.
